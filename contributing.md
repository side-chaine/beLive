# Contributing to beLive

We welcome contributions to the beLive project! Please read this guide to understand how you can contribute.

## How to Contribute

1.  **Fork the repository:** Start by forking the [beLive repository](https://github.com/side-chaine/beLive) to your GitHub account.
2.  **Clone your fork:** Clone your forked repository to your local machine.
    ```bash
    git clone https://github.com/YOUR_USERNAME/beLive.git
    ```
3.  **Create a new branch:** Create a new branch for your feature or bug fix.
    ```bash
    git checkout -b feature/your-feature-name
    ```
4.  **Make your changes:** Implement your changes, adhering to the project's code style and guidelines.
5.  **Run demo locally:** To run the demo locally, open `index.html` in your browser. For more advanced demos, refer to the `test/` directory.
6.  **Commit your changes:** Write clear and concise commit messages.
    *   **DCO Sign-off:** All commits must be signed off. This certifies that you either wrote the contribution or have the right to submit it under the project's license. To do this, add `--signoff` (or `-s`) to your commit command:
        ```bash
        git commit -s -m "feat: your commit message"
        ```
7.  **Push your branch:** Push your new branch to your forked repository.
    ```bash
    git push origin feature/your-feature-name
    ```
8.  **Create a Pull Request:** Open a pull request from your fork to the `main` branch of the main beLive repository. Please fill out the [pull request template](.github/PULL_REQUEST_TEMPLATE.md) completely.
