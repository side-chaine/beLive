   - **Фикс:** заменён на `for (let idx = currentIdx + 1; idx < maxSearchIdx; idx++)` — break на первом containment ≥ 0.8.

2. **NOT MAPPED блоки не фильтровались** — CGP вычислял containment для Intro на MJ (score=0.500), хотя спецификация FLCG/BVP требует `if (startIdx == null) continue`.

3. **Не та структура MJ** — синтетическая фикстура с блоками, сгруппированными по типу (все Verse подряд, все Chorus подряд), вместо хронологической структуры реальной песни (Intro→V1→C1→V2→C2→PC1→V3→C3→PC2→B1→V4→C4→Solo→B2→V5→C5).

## Часть 2: Концептуальная карта — Basket Quality Control (Корзинки)

### Метафора (утверждена)

> **Genius-блоки = корзинки с наклейками**
> На каждой корзинке написан точный список продуктов (строки).
>
> **LRC-строки = продукты на складе**
>
> **Текущий алгоритм (DP) = робот-упаковщик, который НЕ ЧИТАЕТ наклейки**
> Он просто нюхает продукты и кладёт их в корзинку, где пахнет похоже.
>
> **Нужный слой = контролёр качества**
> После упаковки проверяет: первая строка в корзинке Outro совпадает
> с первой строкой на наклейке Outro? Нет — брак, переложи!

### Проблема (TC-155, verified)

```
Genius Outro, строка 1: "I wanna run away and open up my mind"
Genius Chorus 3, строка 4: "I wanna shut the door, and open up my mind"

LRC[54] = "And open up my mind"  ← хвост Chorus 3, ЛЕЗЕТ в Outro
LRC[55] = "I wanna run away and open up my mind"  ← настоящее начало Outro

DP выбрал LRC[54] вместо LRC[55] из-за:
1. WORDS_WINDOW=3: LRC[54] видит LRC[55,56] в окне → Jaccard=1.0 (накрутка)
2. Gap penalty: LRC[54] ближе к ожидаемому gap от Chorus 3
```

### Решение (концепт — требует архитектурной проработки)

**Идея:** Добавить слой верификации границ блоков ПОСЛЕ DP.

Слой должен проверять:
1. Первая строка блока в LRC совпадает с первой строкой блока в Genius?
   - Если ДА → всё ок
   - Если НЕТ → найти LRC-строку с rawScore=1.0 (exact match) и пододвинуть startIdx

2. Нижняя строка блока не должна быть первой строкой СЛЕДУЮЩЕГО блока
   - Проверка: строка N не совпадает со строкой 1 соседнего блока? 
   - Если совпадает — отрезать

**Где внедрять:** Между Section 3 (backward trace DP) и Section 4 (range assignment).
Новый Pass 2.5. Не трогает сам DP, только корректирует его результат.

### Риски

- Новая эвристика поверх эвристики — может создать каскад (правило с хэндоффа)
- Нужен полный candidate dump ДО внедрения (уже сделан)
- Нельзя переносить решение из TC-153 один-в-один — другой код-путь
